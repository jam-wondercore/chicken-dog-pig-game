# 功能需求

## 1. 頁面結構

### Tab 切換系統
- 設定頁（Setup Page）
- 遊戲頁（Game Page）
- 使用 Tab 切換，兩個頁面分開呈現

## 2. 設定頁功能

### 2.1 組別管理
- 可自由新增組別（無數量限制）
- 可刪除現有組別
- 可切換不同組別進行編輯
- 每組固定 2x4 網格（2行 x 4列 = 8 張圖片）

### 2.2 圖片上傳
- **批次上傳**：一次選擇多張圖片，自動填入當前組別的空格子
- **個別替換**：點擊單一格子可上傳/替換該格子的圖片
- 圖片即時預覽顯示
- 已上傳的格子顯示縮圖
- 未上傳的格子顯示提示文字

### 2.3 資料管理
- 清除所有組別與圖片資料
- 開始遊戲按鈕（切換到遊戲頁）

## 3. 遊戲頁功能

### 3.1 遊戲資訊
- 顯示當前組別編號（第 X / 總共 Y 組）
- 顯示播放進度（當前格子 Z / 8）

### 3.2 遊戲播放（無互動）
- 2x4 圖片網格顯示
- **外框跟隨音樂節奏自動播放**：外框會自動依序移動到每個格子，配合音樂節奏
- **純觀看模式**：不需要任何點擊，外框自動播完 8 個格子
- 單組播放完成後自動切換到下一組
- 所有組播放完畢後可選擇重新開始或回到設定頁

### 3.3 遊戲控制
- 暫停/繼續遊戲
- 回到設定頁
- 音樂播放狀態顯示

## 4. 音樂系統

### 音樂播放
- 使用預設音樂檔案：`game.mp3`
- 循環播放（播完自動重頭播放）
- 不提供使用者上傳音樂功能
- 音樂隨遊戲狀態播放/暫停

## 5. 資料儲存

### 儲存機制
- **不持久化**：不使用 LocalStorage 或後端儲存
- 重整頁面後所有資料清空
- 所有圖片暫存於記憶體中（使用 Data URL）
